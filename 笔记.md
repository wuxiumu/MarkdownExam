# MarkdownExam 高赞项目计划文档

## 0. 项目定位

MarkdownExam 是一个面向学习者、知识博主、技术团队和课程作者的轻量级考试与知识测评系统。项目核心特点是：**用 Markdown 管理题库，用原生前端完成考试体验，用 PHP 提供轻量接口，不依赖数据库和复杂框架**。

项目目标不是做一个“大而全”的考试平台，而是做一个足够轻、足够好用、足够容易二次开发的开源项目。它应该做到下载即用、结构清晰、体验完整、文档友好，适合在 GitHub、掘金、知乎、B站等平台展示和传播。

## 一、概要计划

### 1. 项目目标

1. 打造一个开箱即用的 Markdown 题库考试系统。
2. 支持单选、多选、判断、简答等常见题型。
3. 支持考试模式、练习模式、错题模式和解析模式。
4. 支持明暗主题、移动端适配、本地进度保存。
5. 保持技术栈极简：HTML、CSS、原生 JavaScript、PHP、Markdown。
6. 提供完整 README、示例题库、部署教程和二次开发说明。

### 2. 核心卖点

- **Markdown 即题库**：非技术用户也能维护题目，复制、编辑、版本管理都方便。
- **无数据库部署**：降低部署门槛，适合个人项目、课程资料、内部培训。
- **前后端分离**：前端只负责交互展示，后端只负责解析和返回数据。
- **轻量可读源码**：适合作为入门项目、教学项目和二次开发模板。
- **完整考试体验**：计时、判分、解析、错题、进度、主题、响应式布局都具备。
- **内容驱动传播**：可以内置 AI、前端、后端、考研、软考等示例题库，方便演示。

### 3. 目标用户

| 用户类型 | 使用场景 | 核心诉求 |
| --- | --- | --- |
| 学习者 | 自测、刷题、复习 | 简单、好用、可离线 |
| 知识博主 | 发布知识测验 | 易分享、可定制 |
| 技术团队 | 内部培训测评 | 轻部署、可维护 |
| 教师/课程作者 | 课程配套测验 | 题库编辑方便 |
| 开发者 | 学习前后端分离项目 | 结构清晰、易扩展 |

### 4. 技术路线

| 层级 | 技术选型 | 说明 |
| --- | --- | --- |
| 前端 | HTML + CSS + 原生 JavaScript | 不使用框架，降低学习和运行成本 |
| 后端 | 原生 PHP | 负责 Markdown 读取、解析、JSON 输出 |
| 数据 | Markdown 文件 | 题库、人群、标签、答案、解析统一维护 |
| 存储 | localStorage | 保存主题、答题进度、错题记录 |
| 部署 | PHP 内置服务器 / Nginx / Apache | 本地和服务器均可运行 |

### 2. 前端模块

#### 2.1 页面骨架模块

负责页面基础布局，包括顶部栏、考试信息区、题目区、答题卡、底部操作栏和结果区。

核心区域：

- 顶部栏：项目名、模式切换、主题切换。
- 考试信息：题库名、题目数量、当前进度、剩余时间。
- 题目区：题干、选项、解析、收藏按钮。
- 答题卡：题号状态、快速跳转。
- 操作区：上一题、下一题、提交、重新开始。
- 结果区：得分、正确率、错题列表、用时统计。

#### 2.2 题目渲染模块

负责把后端返回的 JSON 数据渲染成可交互题目。

支持题型：

- 单选题：radio。
- 多选题：checkbox。
- 判断题：true/false 按钮。
- 简答题：textarea。

渲染规则：

- 每道题必须显示题号、题干、题型。
- 选项需要保持 Markdown 中的顺序。
- 未提交前默认不显示答案。
- 练习模式可以答完立即显示解析。
- 考试模式交卷后统一显示解析。

#### 2.3 考试状态模块

负责管理考试过程中的状态。

核心状态：

```javascript
const examState = {
  mode: 'exam',
  currentIndex: 0,
  questions: [],
  answers: {},
  marked: [],
  wrongQuestions: [],
  startTime: null,
  endTime: null,
  duration: 0,
  submitted: false
};
```

#### 2.4 判分模块

负责对比用户答案与标准答案，生成考试结果。

判分规则：

- 单选题：用户答案等于标准答案，得满分。
- 判断题：用户答案等于标准答案，得满分。
- 多选题：选项完全一致才得分，漏选、错选不得分。
- 简答题：默认不自动判分，只展示参考答案；后续可扩展关键词判分。

结果指标：

- 总题数。
- 已答题数。
- 正确题数。
- 错误题数。
- 正确率。
- 总用时。
- 各题型正确率。

#### 2.5 本地存储模块

负责保存用户体验相关数据。

存储内容：

- 当前主题。
- 最近一次考试进度。
- 最近一次答案。
- 错题记录。
- 收藏题目。
- 用户偏好的考试模式。

建议 localStorage key：

```text
markdown_exam_theme
markdown_exam_progress
markdown_exam_wrong_questions
markdown_exam_favorites
markdown_exam_mode
```

### 3. 后端模块

#### 3.1 Markdown 读取模块

职责：

- 检查题库文件是否存在。
- 读取 Markdown 内容。
- 处理文件读取失败。
- 返回统一错误结构。

#### 3.2 Markdown 解析模块

职责：

- 解析题库标题。
- 解析题目分组。
- 解析题型。
- 解析题干。
- 解析选项。
- 解析答案。
- 解析解析内容。
- 解析标签和难度。

#### 3.3 JSON 输出模块

统一返回格式：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "paper": {
      "title": "前端基础考试",
      "description": "用于测试 HTML、CSS、JavaScript 基础知识",
      "total": 20
    },
    "questions": []
  }
}
```

异常返回格式：

```json
{
  "code": 1001,
  "message": "题库文件不存在",
  "data": null
}
```

#### 3.4 安全模块

基础安全要求：

- 不允许通过接口读取任意路径文件。
- 题库文件路径使用白名单。
- 输出 JSON 时保留中文。
- 前端渲染题干和解析时避免直接插入危险 HTML。
- 后端接口只返回必要数据。

### 4. 数据模块

#### 4.1 Markdown 题库格式

建议使用可读性强、便于解析的格式：

```markdown
# 前端基础考试

> 描述：用于测试 HTML、CSS、JavaScript 基础知识
> 时长：30
> 标签：前端,基础,JavaScript

## 单选题

### Q1
题型：single
难度：easy
标签：HTML
题目：HTML 的全称是什么？

选项：
A. Hyper Text Markup Language
B. High Text Machine Language
C. Hyperlinks Text Mark Language
D. Home Tool Markup Language

答案：A

解析：HTML 是 Hyper Text Markup Language 的缩写，中文通常称为超文本标记语言。

## 多选题

### Q2
题型：multiple
难度：medium
标签：CSS
题目：以下哪些属性会影响盒模型尺寸？

选项：
A. width
B. padding
C. border
D. color

答案：A,B,C

解析：width、padding、border 都会影响盒模型尺寸，color 只影响文本颜色。
```

```json

  "tags": ["HTML"],
  "title": "HTML 的全称是什么？",

    { "key": "B", "text": "High Text Machine Language" }

}
```

## 三、详细设计计划

### 1. 页面交互设计

#### 1.1 首页

首页直接进入考试系统，不做营销式落地页。

需要展示：

- 题库标题。
- 题库描述。
- 题目数量。
- 预计用时。
- 模式选择：考试模式、练习模式。
- 开始按钮。

交互要求：

- 用户点击开始后进入第一题。
- 如果存在未完成进度，提示继续上次进度或重新开始。
- 主题状态从 localStorage 读取。

#### 1.2 考试页

页面布局：

- 左侧或主体：题目内容。
- 右侧或底部：答题卡。
- 顶部：考试信息和倒计时。
- 底部：上一题、下一题、提交。

状态反馈：

- 当前题高亮。
- 已答题显示完成状态。
- 未答题显示默认状态。
- 标记题显示收藏或标记状态。
- 提交前检查是否有未答题。

#### 1.3 结果页

结果页展示：

- 总分。
- 正确率。
- 用时。
- 正确题数和错误题数。
- 错题列表。
- 每道题的用户答案、正确答案、解析。
- 重新考试按钮。
- 只看错题按钮。

### 2. 题库解析设计

#### 2.1 解析流程

1. 读取 Markdown 文件。
2. 标准化换行符。
3. 提取一级标题作为试卷标题。
4. 提取引用块中的元信息。
5. 按 `### Q数字` 分割题目。
6. 对每个题块提取字段。
7. 校验必填字段。
8. 转换为统一 JSON。
9. 返回给前端。

#### 2.2 字段校验

必填字段：

- 题型。
- 题目。
- 答案。
- 解析。

选择题必填字段：

- 选项。
- 至少两个选项。
- 答案必须在选项 key 中存在。

错误处理：

- 缺少题型：返回题号和错误原因。
- 缺少答案：返回题号和错误原因。
- 答案不存在于选项中：返回题号、答案、可选项。
- 文件不存在：返回明确错误码。

### 3. 前端状态设计

#### 3.1 初始化

流程：

1. 加载主题。
2. 请求题库接口。
3. 初始化考试状态。
4. 检查本地进度。
5. 渲染首页或继续考试提示。

#### 3.2 答题

流程：

1. 用户选择答案。
2. 更新 `examState.answers`。
3. 保存进度到 localStorage。
4. 更新答题卡状态。
5. 练习模式下显示即时反馈。

#### 3.3 交卷

流程：

1. 检查未答题数量。
2. 弹出确认提示。
3. 计算得分。
4. 生成错题记录。
5. 保存错题本。
6. 清理考试进度。
7. 渲染结果页。

### 4. UI 设计计划

#### 4.1 视觉风格

关键词：

- 清爽。
- 高可读。
- 工具感。
- 轻量。
- 专注学习。

配色建议：

- 主色：蓝色或绿色，用于按钮和进度。
- 背景：浅灰和白色组合。
- 深色模式：深灰背景，避免纯黑。
- 正确状态：绿色。
- 错误状态：红色。
- 标记状态：黄色。

#### 4.2 响应式设计

桌面端：

- 主内容区和答题卡左右布局。
- 题目区宽度优先。
- 答题卡固定宽度。

移动端：

- 单列布局。
- 答题卡收起为底部弹层或横向滚动条。
- 操作按钮固定在底部。
- 字体和按钮尺寸适合触摸操作。

### 8. 风险与应对

| 风险 | 表现 | 应对 |
| --- | --- | --- |
| Markdown 格式不规范 | 解析失败 | 制定严格题库格式并给出错误提示 |
| 原生 JS 文件变大 | 维护困难 | 按模块拆分文件 |
| 无数据库能力有限 | 难以做多用户成绩 | 当前版本聚焦单机轻量，后续扩展可选数据库 |
| 简答题自动判分不准 | 误判 | V1.0 先展示参考答案，不做自动判分 |
| 移动端体验差 | 答题不方便 | 早期就加入响应式布局 |

## 四、当前下一步

建议从 V0.1 开始落地，先完成一个最小可运行版本：

1. 建立项目目录。
2. 编写默认 `questions.md`。
3. 编写 `api/questions.php`。
4. 编写 `index.html`。
5. 编写 `css/style.css`。
6. 编写 `js/app.js`。
7. 本地启动 PHP 服务验证完整链路。

V0.1 的完成标准只有一个：**用户打开页面后，可以加载 Markdown 题库，完成答题，提交后看到分数和解析**。
