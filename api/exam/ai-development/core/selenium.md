### Q1
题型：single
难度：easy
标签：工具定位
题目：Selenium 主要用于什么？

选项：
A. 自动化控制浏览器并进行 Web 测试
B. 训练深度学习模型
C. 管理本地大语言模型
D. 生成数据库表结构

答案：A

解析：Selenium 是经典浏览器自动化工具，常用于端到端测试和跨浏览器验证。

### Q2
题型：single
难度：normal
标签：WebDriver
题目：Selenium WebDriver 的作用是什么？

选项：
A. 通过统一接口驱动真实浏览器
B. 替代所有后端 API
C. 训练 NLP 模型
D. 压缩图片

答案：A

解析：WebDriver 定义了自动化浏览器的接口，让测试脚本可以操作页面元素和浏览器行为。

### Q3
题型：multiple
难度：normal
标签：测试行为
题目：Selenium 测试通常会模拟哪些用户行为？

选项：
A. 点击
B. 输入
C. 页面跳转
D. 表单提交

答案：A,B,C,D

解析：端到端测试关注真实用户流程，因此会模拟常见交互行为。

### Q4
题型：single
难度：normal
标签：稳定性
题目：Selenium 测试不稳定的常见原因是什么？

选项：
A. 页面异步加载和等待条件处理不当
B. 浏览器不能显示网页
C. 测试永远不需要等待
D. HTML 没有任何结构

答案：A

解析：Web 页面常有异步请求和动态渲染，测试需要显式等待合适状态。

### Q5
题型：single
难度：hard
标签：选型
题目：Selenium 的核心优势更偏向哪一类？

选项：
A. 成熟生态和跨浏览器兼容性
B. 只适合训练神经网络
C. 完全不需要浏览器驱动
D. 只能测试命令行程序

答案：A

解析：Selenium 历史成熟，支持多语言和多浏览器，是 Web 自动化测试的重要工具。
