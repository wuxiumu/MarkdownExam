### Q1
题型：single
难度：easy
标签：核心概念
题目：在 AI Agent 场景中，Skill 最接近什么？

选项：
A. 可复用的领域流程、规则和工具使用说明
B. 随机生成的一段闲聊文本
C. 完全替代模型的数据库
D. 只用于改变界面颜色的配置

答案：A

解析：Skill 用来把稳定的专业流程和约束沉淀下来，让 Agent 在特定任务上更一致。

### Q2
题型：single
难度：normal
标签：适用场景
题目：什么时候最适合把经验沉淀为 Skill？

选项：
A. 某类任务反复出现，并且有稳定步骤和注意事项
B. 任务只做一次且完全不可复用
C. 需求还没有任何规律
D. 只是想让提示词更长

答案：A

解析：Skill 的价值来自复用。反复出现、流程稳定、容易出错的任务最适合沉淀。

### Q3
题型：multiple
难度：normal
标签：内容结构
题目：一个实用 Skill 通常应该说明哪些内容？

选项：
A. 何时使用
B. 操作步骤
C. 质量标准或验收方式
D. 禁止事项和风险边界

答案：A,B,C,D

解析：好的 Skill 不只是告诉模型做什么，也要说明什么时候做、怎么做、做到什么程度以及不能做什么。

### Q4
题型：single
难度：normal
标签：工程价值
题目：Skills 对团队协作最直接的价值是什么？

选项：
A. 把团队约定变成可复用的执行规范
B. 让每次输出都完全随机
C. 取消代码评审
D. 替代所有测试

答案：A

解析：Skill 可以让 Agent 更稳定地遵守团队的工程规范、文档格式和交付流程。

### Q5
题型：single
难度：hard
标签：边界
题目：为什么 Skill 不应该写成一本百科全书？

选项：
A. Skill 应聚焦可执行流程，过多背景会稀释关键指令
B. 模型不能阅读任何文字
C. Skill 越长一定越准确
D. Skill 只能包含代码

答案：A

解析：Skill 要服务执行。高密度、低歧义、可操作，比堆砌大量背景更有效。
