### Q1
题型：single
难度：easy
标签：工具定位
题目：Claude Code 更接近哪类工具？

选项：
A. 运行在开发环境中的智能编码代理
B. 只用于画流程图的工具
C. 只用于训练 GPU 的框架
D. 完全离线的文本编辑器

答案：A

解析：Claude Code 的核心是帮助开发者在代码库中阅读、修改、运行和协作完成工程任务。

### Q2
题型：single
难度：normal
标签：上下文
题目：在大型项目中使用 Claude Code，为什么要提供明确上下文？

选项：
A. 帮助模型理解业务目标、代码边界和现有约定
B. 让模型忽略项目结构
C. 让任务变得更模糊
D. 避免任何验证

答案：A

解析：AI 编码工具需要上下文来减少误改，尤其是大型代码库中的架构、约定和测试方式。

### Q3
题型：multiple
难度：normal
标签：协作方式
题目：使用 Claude Code 这类工具时，哪些做法更稳妥？

选项：
A. 先让它解释计划或影响范围
B. 分阶段提交小任务
C. 运行相关测试或检查
D. 一次要求它重写整个系统且不审查

答案：A,B,C

解析：小步协作、明确范围和验证结果，是降低 AI 编码风险的基本方法。

### Q4
题型：single
难度：normal
标签：适用任务
题目：Claude Code 特别适合哪类开发场景？

选项：
A. 需要结合代码库上下文的修改、解释、重构和排错
B. 不需要任何代码上下文的随机问答
C. 永远不运行命令的纯理论讨论
D. 替代所有产品决策

答案：A

解析：代码代理的优势在于能结合项目文件和开发工具链处理真实工程任务。

### Q5
题型：single
难度：hard
标签：风险控制
题目：为什么不能把 Claude Code 的输出直接当作最终事实？

选项：
A. 它仍可能产生错误推理、过度修改或遗漏隐含需求
B. 因为 AI 不能读代码
C. 因为代码不需要验证
D. 因为所有项目都完全相同

答案：A

解析：AI 输出是候选方案，不是最终质量保证。开发者仍需审查、测试和承担发布责任。
