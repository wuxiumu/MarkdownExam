### Q1
题型：single
难度：easy
标签：核心概念
题目：AI Agent 与普通聊天机器人的关键区别更接近哪一项？

选项：
A. Agent 会围绕目标使用工具、执行步骤并根据结果调整
B. Agent 只能回答固定 FAQ
C. Agent 不需要上下文
D. Agent 不能访问任何外部工具

答案：A

解析：Agent 的重点不是单轮回答，而是目标驱动、工具调用、状态管理和迭代执行。

### Q2
题型：multiple
难度：normal
标签：组成部分
题目：一个典型 AI Agent 系统通常包含哪些核心部分？

选项：
A. 模型
B. 工具
C. 上下文或记忆
D. 执行与反馈循环

答案：A,B,C,D

解析：模型负责推理，工具负责行动，上下文保存状态，执行反馈循环让 Agent 能持续修正。

### Q3
题型：single
难度：normal
标签：工具调用
题目：Agent 调用工具时，最重要的工程原则是什么？

选项：
A. 明确工具能力、输入输出和权限边界
B. 让工具描述越模糊越好
C. 永远允许工具执行所有命令
D. 不需要处理工具失败

答案：A

解析：工具边界越清晰，Agent 越容易正确选择和安全使用工具，也更容易调试。

### Q4
题型：single
难度：normal
标签：风险
题目：AI Agent 最大的工程风险之一是什么？

选项：
A. 自主执行错误动作并放大影响
B. 只能处理文本
C. 永远不会犯错
D. 不会产生任何成本

答案：A

解析：Agent 能行动，所以错误不只停留在回答层面，可能影响文件、数据、权限和外部系统。

### Q5
题型：single
难度：hard
标签：设计目标
题目：设计 Agent 时，为什么要建立“观察-计划-执行-反馈”的闭环？

选项：
A. 让系统根据真实结果持续修正，而不是一次性猜完
B. 为了让流程更复杂
C. 为了避免使用任何工具
D. 为了让模型忽略用户目标

答案：A

解析：闭环让 Agent 能把行动结果重新纳入上下文，逐步逼近目标并降低盲目执行。
